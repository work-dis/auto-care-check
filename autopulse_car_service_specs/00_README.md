# AutoPulse — пакет ТЗ для приложения контроля обслуживания автомобиля

**Рабочее название:** AutoPulse. Его можно переименовать без изменения логики.

## Что это за продукт

Responsive web/PWA-приложение для одного владельца или семьи, которое помогает не забывать о техническом обслуживании автомобиля. Пользователь добавляет автомобиль и актуальный пробег, ведёт историю работ и расходов, видит на главном экране понятное «цифровое табло»: что просрочено, что скоро нужно сделать, что стоит проверить и что пока в норме.

Это **не диагностический сервис** и не должен заявлять, что реально определяет техническое состояние машины. Статусы строятся только на введённых данных: дате, пробеге, установленном интервале, отмеченной пользователем проблеме и истории обслуживания.

## Главная ценность

1. Один экран вместо заметок, чатов с мастером и забытых чеков.
2. Владелец сразу понимает приоритет: «сделать срочно», «запланировать», «наблюдать», «пока не трогать».
3. Напоминания можно задавать по дате, пробегу или одновременно по обоим условиям.
4. История не теряется: старые работы, расходы и использованные запчасти сохраняются даже после изменения плана обслуживания.

## Как использовать этот пакет

1. Передайте нейросети файл `08_MASTER_PROMPT_FOR_AI.md`.
2. Передайте ей остальные `.md`-файлы как техническую документацию и источник истины.
3. Для поэтапной разработки используйте `09_ITERATION_PROMPTS.md` — по одному промпту на итерацию, с проверкой результата после каждой.
4. Не разрешайте нейросети переписывать бизнес-логику «по своему усмотрению»: статусы, напоминания, архивирование и история работ должны соответствовать этому ТЗ.

## Рекомендуемая техническая основа

Рекомендация для самостоятельной разработки и удобной дальнейшей эксплуатации:

- **Frontend:** Next.js / React, TypeScript, Tailwind CSS, компонентная библиотека уровня shadcn/ui, Lucide icons.
- **Backend:** TypeScript API на Fastify либо серверные маршруты Next.js. Выбрать один подход и не дублировать бизнес-логику.
- **Валидация:** Zod на границах API и форм.
- **База данных:** PostgreSQL + Prisma.
- **Фоновые задачи:** отдельный worker/cron для напоминаний; не выполнять массовую отправку уведомлений в HTTP-запросе пользователя.
- **Инфраструктура:** Docker Compose для локальной и серверной среды; переменные окружения; миграции БД; резервное копирование PostgreSQL.
- **Клиент:** адаптивный web/PWA. На мобильном устройство должно быть удобно добавить пробег одной рукой.

Допускается адаптировать стек под уже существующий репозиторий, но нельзя ухудшать требования по типизации, валидации, истории данных и работе напоминаний.
